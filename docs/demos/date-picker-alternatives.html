方案 C · 推荐方向之一

纵向滚动 · 连续月列

可替换：日程 Tab 顶栏、Sheet 内选日、任务改期 Popover

↓ 在此区域双指上下滑，逐月 snap

优点

  • 双指纵向滑动自然翻月，符合触控板习惯
  • 可连滑多个月，远日期比点箭头快
  • 与「日程列表纵向滚动」心智一致

缺点

  • 需自研 SwiftUI 组件（或用 ScrollView + snap）
  • 一次只见 ~1 个月，不如 grid 直观
  • 要处理初始 scroll 定位、选中态同步
适用 日程浏览 + 选日统一控件；改期 Popover。★★★★☆
方案 D

横向滑动 · 月卡 Carousel

可替换：`LearningScheduleView` 顶栏 chevron 翻月

← → 双指横向滑，或拖动下方滚动条

优点

  • 横向滑月像 iOS 日历 App，视觉愉悦

方案 G · 交互模式（非样式）

Menu 内嵌 vs 独立 Popover

对应：`LearningTaskRow` 里 Menu { DatePicker } — 当前最差场景

❌ 现状：Menu 里塞日历
推迟到明天
选择日期
删除

窄、难点的迷你日历，几乎无法手势操作

✅ 建议：Menu 只触发 Popover

Popover 分离的优点

  • 日历有足够宽高，可接滚轮/滑动翻月
  • Menu 保持轻量（文字动作 +「改期…」入口）
  • SwiftUI：`.popover` / `.sheet(item:)` 比 Menu 内 DatePicker 可控

缺点

  • 多一次点击（Menu → 改期）
  • 需管理 popover 锚点与 dismiss
结论 无论选哪种日历样式，都应先把 DatePicker 移出 Menu。这是零样式改动也能立刻变好的步骤。★★★★★ 优先级 P0
