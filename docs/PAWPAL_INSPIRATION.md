# PawPal 借鉴头脑风暴

> 参考项目：[PawPal](https://github.com/zebangeth/PawPal)（Electron/React 桌面宠物，2026）
> 整理时间：2026-05-02｜更新：2026-05-03（补充帧动画、休息活动提示）

---

## 一、背景对比

| 维度 | MalDaze | PawPal |
|---|---|---|
| 技术栈 | Swift 5 + SwiftUI + AppKit | Electron + React 19 + TypeScript |
| 平台 | macOS 原生 | macOS（跨平台潜力） |
| 核心打断机制 | **霸屏**：全屏渐暗 + 小狗居中，强制休息 | **跑屏**：小狗横穿整个屏幕，不完全遮挡 |
| 提醒类型 | 护眼休息（番茄 / 整点锚点） | 休息 + **喝水** + 专注守卫（三类） |
| AI 功能 | Gemini 自然语言 → EventKit 提醒 | 无 |
| 宠物皮肤 | 单一线条小狗（4 种颜色状态） | 线条小狗 + 金毛 puppy（两套） |
| 国际化 | 代码内中英混用，无正式 i18n | 中 / 英文完整切换 |
| 专注检测 | 无 | 检测当前前台 App，刷社交媒体时提醒 |
| 本地统计 | 无 | electron-store 记录使用数据 |

## 二、可借鉴的具体方向

### 2.1 喝水提醒（独立提醒维度）

**PawPal 的做法**：与休息提醒完全独立的一个提醒通道，有自己的时间间隔设置。

**MalDaze 可以做什么**：

目前 MalDaze 只有一个"独立倒计时提醒"（原名 7-minute reminder），时长 1–180 分钟，用途完全由用户自定义。可以在此基础上增加**快捷预设**，而不必新建一套引擎：

- 在控制面板 `SevenMinuteReminderShortcut` 旁加「喝水」快速按钮，一键设置 45 分钟倒计时，触发时气泡文案换成「该喝水了」。
- 如果产品上想独立，也可以在设置里增加「喝水间隔」独立开关，走自己的 `ManualTimerEngine` 实例。

**关键差异**：PawPal 的喝水提醒与休息提醒完全平行；MalDaze 复用现有计时引擎成本更低，且支持 AI 提醒写入 EventKit，深度更强。

---

### 2.2 跑屏动画（非霸屏的轻量打断模式）

**PawPal 的做法**：休息到期时，小狗从屏幕一侧运动到另一侧，穿过整个屏幕，视觉上足够显眼，但**不遮挡内容、不拦截点击**。

**MalDaze 可以做什么**：

作为现有霸屏的**补充模式**（不是替代）：

```
设置 → 打断强度
  ◉ 强（默认）：全屏渐暗 + 小狗居中
  ○ 轻：小狗跑过屏幕，不遮挡
```

**实现要点**（Swift/AppKit）：
- 小狗窗口（`NSWindow.Level.floating`）在屏幕外出现，用 `NSAnimationContext` 或 `CAAnimation` 驱动 `x` 坐标从 `-width` 到 `screenWidth + width`。
- 整个动画约 3–4 秒，结束后窗口回到常驻桌宠位置。
- 不需要修改 `WindowManager` 的霸屏逻辑，只需在 `PetStageView` 里增加跑步动画序列。

**为什么值得做**：对于不喜欢被打断工作流的用户（例如正在打字、看视频），轻量模式可以大幅降低卸载率，同时保留桌宠存在感。

---

### 2.3 专注守卫（前台 App 检测）

**PawPal 的做法**：监测用户当前使用的 App，如果是 Twitter、微博、抖音等社交媒体，小狗主动出现提醒。依赖 Accessibility 权限。

**MalDaze 可以做什么**：

这与 MalDaze 的"护眼"定位有天然延伸空间——**护眼是保护视力，护专注是保护时间**，两者都是"自我保护"。

具体实现思路：
- 用 `NSWorkspace.shared.frontmostApplication` 轮询（每 10–30 秒）或监听 `NSWorkspace.didActivateApplicationNotification`。
- 用户可在设置中维护一个「分心 App 名单」（黑名单，Bundle ID 列表）。
- 触发时：小狗做一个小动作（抬头摇尾、气泡文字），而不是霸屏——保持轻量。
- **不需要 Accessibility 权限**：`NSWorkspace` API 可以读取前台 App，无需辅助功能授权；只有"读取其他 App 的 UI 元素"才需要。

**风险**：需要向用户解释"为什么要监控正在用的 App"，隐私敏感度较高。建议默认关闭，Settings 里配置，文案强调"所有数据留本机"。

---

### 2.4 多宠物皮肤

**PawPal 的做法**：提供线条小狗和金毛两种外观，用户可切换。

**MalDaze 现状**：`PetDisplayMode` 描述的是**状态**（红/黑/白描边/思考），不是**皮肤**。目前只有一套线条小狗图形。

**MalDaze 可以做什么**：

区分两个层次：

1. **配色主题**（低成本）：不换形状，只换颜色方案。例如"夜间模式小狗"（深色背景 + 白线）vs 现有"白天模式"。与 `PetDisplayMode` 正交，只是 Color Token。

2. **全皮肤切换**（高成本）：真正换图形资产，例如加一套猫咪或熊猫。需要为每套皮肤准备对应 4 种状态（休息/运行/暂停/思考）的图形。`PetRenderer` 已经是独立模块，可以在此基础上引入 `PetSkin` enum 注入不同图形提供者。

**优先推荐低成本路线**：先做配色主题切换，快速给用户"可定制感"，收集反馈后再决定是否做多形象。

---

### 2.5 渐进式打断（run → dim → fullscreen 三级）

**PawPal 的做法**：只有一级——跑屏。MalDaze 只有一级——全屏霸屏。

**MalDaze 可以演进成**：

```
[阶段 1] 倒计时最后 60 秒：小狗窗口轻微抖动 / 气泡"快休息了"
[阶段 2] 倒计时最后 15 秒：小狗跑屏动画（参考 2.2）
[阶段 3] 计时归零：全屏渐暗霸屏（现有逻辑）
```

好处：给用户"心理预期"，减少被突然霸屏的挫败感；同时对于"正在打字到关键处"的用户，看到阶段 1 / 2 后可以主动快速收尾再接受休息。

实现：在 `TimerEngine` 的 `TimeState` 里暴露 `remainingSeconds`，`WindowManager` 观察这个值并在阈值处触发对应动画。这个 UI 层逻辑不需要改 `TimerEngine` 协议本身。

---

### 2.6 本地使用统计

**PawPal 的做法**：用 `electron-store` 持久化统计数据（具体指标未在 README 详细列出，但代码结构里有 stats 相关内容）。

**MalDaze 可以做什么**：

轻量版统计，写入 `UserDefaults` 或一个本地 JSON 文件：

- 今日完成休息次数 / 跳过次数
- 连续坚持天数（streak）
- 当日专注总时长（番茄模式下）

在控制面板底部或设置页展示。不联网，不上报。

**价值判断**：统计数据是激励坚持使用的重要机制（"已坚持 7 天"的成就感），对留存率有正向影响，实现成本低。

---

### 2.7 国际化（i18n）

**PawPal 的做法**：代码里有完整的 `zh` / `en` 语言切换，所有用户可见字符串都走 i18n key。

**MalDaze 现状**：UI 字符串中英混杂，没有 `.strings` 文件或 `String(localized:)` 体系，目前实际只面向中文用户。

**MalDaze 可以做什么**：

如果有出海打算，引入标准 `Localizable.strings` + `String(localized:)` 体系是必要的，但工作量不小（涉及所有 View 文件）。

**短期建议**：不必现在做完整 i18n，但至少将 AI 提示词（`SmartReminderPromptBuilder`）的语言设定做成可配置，方便将来国际用户用英文与 Gemini 交互。

---

### 2.8 宠物帧动画（让桌宠"活起来"）

**PawPal 的做法**：宠物有独立的帧动画——待机时原地动作（呼吸/眨眼/摇尾），跑屏时有行走循环，受到互动时有反应帧。这是桌宠"有生命感"的核心来源。

**MalDaze 现状**：`PetRenderer` 渲染一张静态图，仅通过 4 种颜色配置区分状态（运行黑 / 休息红 / 暂停白描边 / 思考）。小狗视觉上是静止的。

**MalDaze 可以做什么**：

分两个层次推进，互不依赖：

1. **待机微动画（低成本）**：在常态小窗内，用 `CABasicAnimation` 对 `PetRenderer` 的 layer 做极小幅度的周期性变换——例如每 4 秒做一次轻微的 `scaleY` 呼吸起伏（幅度 ±2%），或左右轻摇（`rotation` ±1°）。不需要新增图形资产，完全用现有单张图 + Core Animation 实现，视觉上能大幅提升"在场感"。

2. **多帧精灵图（中成本）**：为小狗绘制 3–5 帧的待机循环（例如：站立→微蹲→站立→抬头），放入 Assets.xcassets 作为帧序列。`PetRenderer` 增加 `startIdleAnimation()` / `stopIdleAnimation()` 方法，用 `Timer` 或 `CADisplayLink` 驱动帧切换。休息变红时切换到"红色版"同一套帧。

**优先推荐低成本路线**：先用 Core Animation 伪动画快速验证"动起来"的体验提升，收集真实反馈后再决定是否投入多帧绘制。

**为什么值得做**：桌宠与普通状态栏图标最大的区别就是"有生命"。静止的小狗容易被用户视觉忽略，而哪怕 ±2% 的呼吸动画就能在余光中制造存在感，这是留存率的底层驱动力。

### 2.9 休息期间的活动提示

**PawPal 的做法**：休息/提醒触发时，界面会附带具体的活动引导（如"目视远处 20 秒"、"喝杯水"、"活动肩颈"），而不只是一个倒计时数字。

**MalDaze 现状**：霸屏界面 = 渐暗背景 + 居中红色小狗 + 左下角大号倒计时。没有任何文字引导，用户被迫等待，但不知道"该用这 5 分钟做什么"。

**MalDaze 可以做什么**：

在 `PetStageView` 的休息布局中，于倒计时附近添加一行提示文字。文案来源可分两级：

**静态文案库（零成本起步）**：
```
- 目视 6 米外，持续 20 秒
- 闭眼，做 3 次深呼吸
- 站起来，活动肩颈和手腕
- 喝一口水
- 让眼睛完全放松，不看任何屏幕
```
每次休息随机抽取一条，用淡入动画出现，字体小而柔和，不与倒计时争抢注意力。

**AI 生成版（差异化）**：借助 MalDaze 已有的 Gemini 接入，在触发休息时异步生成一条个性化提示，替换静态文案。例如结合当前时间（"已连续工作 50 分钟"）生成更具针对性的建议。

**实现要点**：
- 在 `beginRestCycle` 调用时确定本轮文案（静态版：同步随机抽取；AI 版：异步生成，有默认兜底）。
- 文案以 `NSTextField` 渲染，`alpha` 从 0 淡入，字色白色带阴影，与倒计时样式一致但字号约 18–22pt。
- 休息提前结束（连击 10 下）时随倒计时一同消失，无需特殊处理。

**为什么值得做**：把"被迫等待"变成"被引导休息"，用户的抵触感会明显降低。这也是 MalDaze 相对 PawPal 能做到的差异化——PawPal 的提示是静态规则，MalDaze 的可以是 AI 理解用户当前状态后生成的。

## 四、不适合直接照搬的部分

### Electron 的"总在置顶"窗口策略

PawPal 用 Electron 的 `alwaysOnTop: true` 实现小狗始终可见。MalDaze 已经在用 `NSWindow.Level` 做更精细的层级控制（常规期 floating、霸屏期 screenSaver），macOS 原生能力远超 Electron，**不需要也不应该**照搬 Electron 的做法。

### 社交媒体黑名单的硬编码

PawPal 内置了一批社交媒体 App 的名称列表作为专注守卫的默认黑名单。MalDaze 如果做专注守卫，应设计成**用户可配置**的 Bundle ID 列表，而不是硬编码，避免随着 App 更新和新平台出现而失效。

### 无 AI 的纯规则提醒

PawPal 的提醒全部基于固定时间间隔规则。MalDaze 已有 Gemini AI 支撑的智能提醒，不应退化成纯规则，应在规则提醒的基础上保持 AI 能力的差异化优势。

### 跨平台设计

PawPal 选择 Electron 有部分原因是跨平台潜力（macOS + Windows）。MalDaze 走 Swift 原生路线，与 macOS 深度集成（EventKit、Carbon 快捷键、NSWorkspace）是核心竞争力，不建议为了跨平台放弃原生。

---

## 五、一句话总结

PawPal 最值得 MalDaze 借鉴的不是某个具体功能，而是**"打断不一定要粗暴"**的设计哲学——通过跑屏、喝水、渐进预警等轻量交互，在减少负担感的同时维持存在感。MalDaze 完全可以在保留"强休息"能力的前提下，提供"弱提醒"的补充通道，满足不同工作场景下的用户。
