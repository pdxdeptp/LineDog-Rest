# Round 03 Review: Long-Term Learning Planner

## Reviewer Lens

Learning plans are not all the same. A tutorial repo, LeetCode practice, interview topic prep, project rebuild, and resume packaging need different planning shapes.

## Issues Found

1. The draft compiler was output-driven, but it did not explicitly choose a plan archetype.
2. Recurring practice and review loops were under-specified, especially for LeetCode and interview prep.
3. Completion standards needed to be tied to observable artifacts or review outcomes.

## Modifications Made

- Added plan archetypes to `design.md`: finite learning project, recurring practice, topic review, rebuild/clone, project packaging, and existing-project phase.
- Added `Plan Archetype Selection` requirement to `study-intake-planning`.
- Strengthened daily draft behavior so recurring plans can create cadence and review loops rather than only one-time tasks.

## Result

The design now supports the user's real learning portfolio: repos and courses can become finite projects, while LeetCode and interview prep can become durable cadences with review loops.
