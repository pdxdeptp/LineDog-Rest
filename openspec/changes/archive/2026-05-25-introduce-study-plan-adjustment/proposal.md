## Why

ITEM-001 and ITEM-002 established confirmed study plans and deterministic study views, but the active plan still cannot be safely adjusted after confirmation. The remaining v2 core needs user-initiated changes that are mechanical, inspectable, and factual: unfinished tasks roll forward, user moves cascade within the project, red states are shown honestly, and dialogue adjustments require preview plus apply.

## What Changes

- Add deterministic active-plan adjustment APIs for rolling unfinished tasks, moving tasks with same-project cascade, editing project deadlines, adding tasks, deleting tasks, and managing rest days.
- Extend v2 study view facts so Today, Project Overview, and Calendar can display rolled-task badges, expected-late project state, and over-capacity day state from persisted facts.
- Replace the v1-shaped generic adjustment chat path with a bounded study-plan adjustment preview/apply flow for user-authored natural-language commands.
- Preserve default-mode silence: no assistant suggestion, smart-mode proposal, or cross-project optimization is generated by this change.
- Keep completed project history read-only and preserve project/task facts when adjustment actions complete or archive a project.

## Capabilities

### New Capabilities

- `study-plan-adjustment`: Active study-plan adjustment behavior, including rollover, same-project cascade, deadline edits, red-state facts, task add/delete, rest-day cascade, and dialogue preview/apply.

### Modified Capabilities

- None. Existing v1 specs are left untouched until the planned v2 retirement step; ITEM-002 `study-views` is still an active change and this proposal extends its runtime payloads through the new `study-plan-adjustment` capability rather than editing archived specs.

## Impact

- Backend SQLite schema and migrations for rollover/rest-day metadata.
- Backend query helpers and routers for adjustment commands and red-state facts.
- Backend chat/planner boundary for route-A preview/apply semantics.
- Swift API models/client/protocol for adjustment endpoints and enriched view facts.
- Swift ViewModel and UI for active-plan movement, deadline editing, add/delete, rest-day settings, rolled badges, and adjustment preview/apply.
- Tests across backend, Swift model/client, ViewModel, UI source, and App Use verification.
