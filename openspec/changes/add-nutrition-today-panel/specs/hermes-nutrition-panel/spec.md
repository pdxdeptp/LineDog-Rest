## ADDED Requirements

### Requirement: _refresh_panel 自动维护

Hermes `recommend.py` SHALL 在每次通过 `_update_daily_log` 修改当日数据后调用 `_refresh_panel()`，在同一原子写内更新 `daily_log.json` 的 `panel` 块。

`panel` 的 `consumed`、`remaining`、`targets` SHALL 包含 `sodium_mg`（与现有营养计算一致）。

`panel.suggestions` SHALL remain an empty array `[]` in this first version for schema compatibility only. User-visible nutrition recommendations are superseded by `~/.hermes/data/nutrition/recommendation.json` and MUST NOT be generated by `_refresh_panel`.

#### Scenario: log 后刷新

- **WHEN** 用户通过 `recommend.py log` 成功记录一笔食物
- **THEN** `panel.consumed` 与 `panel.remaining` 反映新累计（含钠）
- **AND** `panel.updatedAt` 更新为当前时间
- **AND** `panel.suggestions` 为空数组

#### Scenario: undo 后刷新

- **WHEN** 用户执行 `recommend.py undo`
- **THEN** `panel` 与撤销后的 `records` 一致

### Requirement: refresh-panel 子命令

Hermes `recommend.py` SHALL 提供 `refresh-panel` 子命令：在不修改 `records` 的前提下重算并写入 `panel`（供晨报、调试、MalDaze 无需 mutating 时的契约修复）。

#### Scenario: 手动 refresh-panel

- **WHEN** 执行 `python3 recommend.py refresh-panel`
- **THEN** `daily_log.json` 的 `panel` 更新且 `records` 不变

### Requirement: integration_smoke panel 契约

`integration_smoke.py` SHALL 包含检查项：读取 `daily_log.json`，断言存在 `panel` 且 `panel.schemaVersion == 1`。

#### Scenario: smoke 全绿

- **WHEN** 执行 `python3 ~/.hermes/scripts/integration_smoke.py`
- **THEN** 营养 panel 检查项 `ok` 为 `true`

### Requirement: 大卡容差 fifty

`panel.calorieSlack` MUST 为 `50`，供 facts/metrics panel 保留兼容语义。

`_refresh_panel` MUST NOT use `plan_engine` to publish user-visible `suggestions` to MalDaze.

#### Scenario: calorieSlack retained

- **WHEN** `recommend.py refresh-panel` updates the panel
- **THEN** `panel.calorieSlack` is `50`
- **AND** `panel.suggestions` is `[]`

### Requirement: v1 Python baseline superseded

v1 Python baseline `panel.suggestions` is superseded by `recommendation.json`. `plan_engine` MAY produce candidate plans for Hermes authoring, but `_refresh_panel` SHALL NOT publish those candidates to `daily_log.panel.suggestions`.

#### Scenario: plan_engine candidate not published

- **WHEN** `plan_engine` can optimize a default candidate set
- **THEN** `refresh-panel` still writes `panel.suggestions` as `[]`
- **AND** any user-visible recommendation must be written through `recommendation.json`

#### Scenario: no recommendation candidate

- **WHEN** 无可用食物或优化失败
- **THEN** `panel.suggestions` 为空数组
- **AND** `consumed` / `remaining` 仍正确写入

### Requirement: 晨报刷新 panel

`morning-briefing.py` 在营养 `auto` 与饮食 facts 段逻辑完成后 SHALL 刷新当日 `panel`（可调用 `refresh-panel` 或内联 `_refresh_panel`）。如果 Morning Briefing 给出用户可见饮食建议，Hermes SHALL additionally write `recommendation.json` through the Hermes-authored recommendation path.

#### Scenario: 晨报后 panel 存在

- **WHEN** 晨报营养段成功执行
- **THEN** `daily_log.json` 含有效 `panel` 且 `panel.schemaVersion == 1`
- **AND** planner-only candidates are not published as fresh recommendations

### Requirement: skill 禁止手改 JSON

`nutrition-menu` skill SHALL 规定：记录与撤销只通过 `recommend.py` 子命令；**禁止**直接编辑 `daily_log.json`。`panel` 由脚本维护；`refresh-panel` 仅供脚本/调试，飞书对话记录仍用 `log`。飞书对话若回复下一步饮食建议，Hermes SHALL write `recommendation.json`; it MUST NOT rely on `panel.suggestions`.

#### Scenario: 飞书记录流程

- **WHEN** 用户在飞书说「吃了燕麦 50g」
- **THEN** agent 调用 `recommend.py log`
- **AND** 不得手改 JSON 补 `panel`
